# バックログアイテム管理仕様

## バックログアイテム粒度
- スクラム推奨レベル（ユーザーストーリー→実装アイテム→テストアイテム）
- 複雑なバックログアイテムはAIが自動分割（スクラム手法に則る）

## バックログアイテム属性
- タイトル、説明、優先度、担当AI、状態、関連スプリント、成果物パス
- ストーリーポイントは進捗可視化用（AIには意味を持たないが人間評価用）

## バックログアイテム状態
- Backlog → Sprint Backlog → In Progress → Review → Done

## バックログアイテム実行
- AIが担当バックログアイテムを自律実行
- 実行結果はgit管理、成果物はGitHubにpush
- レビュー/QAはAIが担当
- 人間はスプリントレビューで成果物評価
